Keywords & variables

Variables

srcdir

srcdir is the directory of source files.

objdir

objdir is the directory of object files.

builddir

builddir is the directory of built executables.

build

build is a list of object files to be built.

dep

dep is a list of dependencies for the program.

all

all is a list of targets included in the all target.

bopt & boptl

bopt (build options) is a list of make variable names for the -D compiler option.
boptl (build options literal) is a list of values for the -D compiler option.

cnames, cnamesf, cdirs, and cdirsf

cnames, and cdirs will be removed by make clean. The ones with f at the end will be removed with -f.

Keywords

include [filename]

include specifies a file in srcdir should be compiled to a .o file and automatically adds it to build.

nextobj [string]

nextobj should be used in front of an include statement, specifying a certain name for the object file rather than the filename of the source.

build [filename]

build will build a file from obj's in build into builddir/[filename], with dependencies in dep

noall

noall will stop the next include, build, and adddir from automatically being added to all.

adddir [directory name]

adddir will create a directory.

nobuild

nobuild will stop the next include from automatically adding obj's to build.

dep [string]

dep will add a dependency to dep.

abuild [filename]

abuild will add a filename from a previous include (usually that has had a noall).

rmbuild [string]

rmbuild will remove a obj previously added by abuild or include or all obj's if [string] is '/' (no quotes).

rmdep [string]

rmdep will remove a dependency from dep added by the dep keyword.

var [variable name] [value]

var will add a make variable [variable name] and assign it to [value] with ':='.

vare [variable name] [value]

vare will add a make variable [variable name] and assign it to [value] with '='.

bopt [macro name] [variable name]

bopt will add a macro in the compiler (with -D) as [macro name], and assign it to [variable name] eg. gcc main.c -D[macro name]=${[variable name]} .

boptl [macro name] [value]

boptl will add a macro in the compiler (with -D) as [macro name], and assign it to [value] eg. gcc main.c -D[macro name]=[value] .

rmbopt [bopt name]

rmbopt will remove a previously defined bopt, specified by a [macro name], in a bopt statement.

rmboptl [boptl name]

rmboptl will remove a previously defined boptl, specified by a [macro name], in a boptl statement.

compiler [string]

compiler will specify the compiler used, or if left blank it will set it to default. (being ${CC} for .c files (or anything else), or ${CXX} for .cpp files.)

linker [string]

linker will specify the linker used, or if left blank it will set it to default. (${CXX})

svar [variable name] [value]

svar will set [variable name] to [value] with ':='

svare [variable name] [value]

svare will set [variable name] to [value] with '='

bname [string]

bname will set builddir to [string]

oname [string]

oname will set objdir to [string]

sname [string]

sname will set srcdir to [string]

cdir [string]

cdir will add [string] to cdirs.

cfile [string]

cfile will add [string] to cnames.

cdirf [string]

cdirf will add [string] to cdirsf.

cfilef [string]

cfilef will add [string] to cnamesf.

make

make will start embedded make, adding any subsequent lines straight to the makefile, until reaching a line with 'end' (no tabs or spaces)

installbin [filename]

installbin will add a install target with builddir/[filename] to be installed to installpath

installpath [path]

installpath will set installpath to [path] (by default /usr/local/bin)

checknext [filename]

checknext will make srcdir/[filename] to be checked (if it's modified) by the next build, include, or adddir
